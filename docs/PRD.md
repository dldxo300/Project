### **[프로젝트명] 나만의 피규어 제작 플랫폼 MVP - 제품 요구사항 명세서 (PRD)**

**1. 개요 (Overview)**

*   **프로젝트 목표:** 고객이 제공하는 단 한 장의 2D 이미지를 AI 기술로 3D 모델링하고, 전문 작가의 수작업을 거쳐 고품질 실물 피규어/미니어처를 제작해주는 주문 제작 중개 플랫폼 MVP를 구축한다.
*   **핵심 가치:**
    1.  **접근성:** 누구나 사진 한 장으로 세상에 하나뿐인 자신만의 굿즈를 소유할 기회를 제공한다.
    2.  **선순환 구조:** 한 번 제작된 주문제작 상품의 3D 데이터를 활용하여 다른 고객이 구매할 수 있는 기성품으로 재판매하는 수익 모델을 검증한다.
*   **MVP 가설 검증:** "사용자들은 AI 기반의 3D 모델링 변환 및 주문 제작 과정에 높은 만족도를 보이며, 이는 유의미한 구매 전환율과 재방문율로 이어질 것이다."
*   **핵심 성공 지표:**
    *   구매 전환율 (주문서 작성 완료 / 총 방문자 수)
    *   사용자 재방문율

**2. 사용자 페르소나 및 핵심 시나리오 (User Personas & Scenarios)**

*   **주요 고객 (20대 대학생, 김민지):** "내가 좋아하는 게임 캐릭터나 나의 반려동물을 내 책상 위에 두고 싶어. 그림은 잘 못 그리지만, 사진 한 장으로 멋진 피규어를 만들어 준다면 바로 주문할 거야."
*   **관리자 (플랫폼 운영자):** "주문 관리부터 작가 매칭, 완성된 작품의 재판매까지 원활하게 관리하여 수익을 극대화하고 싶어."
*   **제작자 (전문 작가):** (MVP에서는 관리자가 직접 소통하거나 지정된 작가와 협업하는 형태로 단순화)

**3. 기능 요구사항 (Functional Requirements)**

**3.1. 사용자 (고객) 기능**

| 기능 분류 | 기능명 | 상세 설명 | 우선순위 |
| :--- | :--- | :--- | :--- |
| **계정** | 회원가입 / 로그인 | Clerk을 이용한 소셜 및 이메일 기반 간편 인증 시스템 | **P0 (필수)** |
| **주문** | **주문제작 의뢰** | - 2D 이미지 업로드 기능<br>- 피규어 사이즈, 추가 요구사항 등을 기입하는 주문서 양식<br>- 주문서 제출 및 접수 확인 | **P0 (필수)** |
| | **작가 매칭 및 소통** | - 주문서 확인 후 관리자가 최적의 작가 매칭 (MVP에서는 자동화 X)<br>- 제작 과정, 수정사항 논의를 위한 1:1 채팅 기능 | **P0 (필수)** |
| | **결제** | - Toss Payments를 연동한 안전한 결제 시스템 (주문 금액, 배송비 포함) | **P0 (필수)** |
| | 주문 내역 확인 | - 나의 주문 목록 (주문 상태: 결제 대기, 제작 중, 제작 완료, 배송 중 등) 확인 | **P1 (높음)** |
| **마켓** | **작품 전시 및 구매** | - 완성된 다른 사람의 주문제작품을 구경하고 구매할 수 있는 마켓<br>- 기성품 상세 페이지 (사진, 가격, 설명) | **P0 (필수)** |
| | 상품 문의 | - 마켓에 등록된 상품에 대한 문의/답변(Q&A) 게시판 기능 | **P1 (높음)** |
| **커뮤니티**| 후기 작성 | - 완성된 작품에 대한 만족도, 별점, 사진을 포함한 후기 작성 기능 | **P1 (높음)** |

**3.2. 관리자 기능**

| 기능 분류 | 기능명 | 상세 설명 | 우선순위 |
| :--- | :--- | :--- | :--- |
| **주문 관리**| **주문서 관리** | - 접수된 모든 주문서 목록 확인 및 상태(확인, 반려 등) 변경<br>- 주문 관련 고객 문의 응대 및 채팅 내역 모니터링 | **P0 (필수)** |
| | 작가 배정 | - 주문서 내용 확인 후 내부 작가에게 작업 할당 | **P0 (필수)** |
| **상품 관리**| **완성 작품 재판매 등록** | - 완료된 주문제작품을 마켓 상품으로 등록<br>- 재판매 가격 책정 (3D 출력 기본가, 도색 추가금 등 옵션 설정)<br>- 상품 정보(이미지, 설명) 입력 및 수정 | **P0 (필수)** |
| | 상품 재고/판매 관리 | - 등록된 마켓 상품의 재고 및 판매 현황 관리 | **P1 (높음)** |
| **회원 관리**| 회원 정보 조회 | - 가입된 회원 목록 및 정보 조회 | **P1 (높음)** |

**4. 기술 스택 (Tech Stack)**

*   **프론트엔드/프레임워크:** Next.js
*   **백엔드/데이터베이스:** Supabase (BaaS)
*   **인증 (Authentication):** Clerk
*   **결제 (Payment):** Toss Payments API
*   **AI 3D 모델링:** (MVP 단계에서는 외부 솔루션 연동 또는 수동 변환 후 결과만 업로드하는 방식 고려. 어떤 AI 엔진을 사용할지는 별도 기술 검토 필요)

**5. MVP 제외 기능 (Out of Scope for MVP)**

*   AI 기반 3D 모델 실시간 미리보기 기능
*   작가용 어드민 페이지 (관리자 페이지에서 역할 대행)
*   정교한 작가 자동 매칭 시스템
*   사용자 간 중고거래 기능
*   다국어 지원 및 해외 결제
